데이터 전처리 공식

모든 ML/DL 문제의 공통 첫 단계

🚀 Step 0 — 라이브러리 임포트 (매번 맨 위에 붙여넣기)
🍳 비유: 요리 전 재료 꺼내기 요리하기 전에 냉장고에서 재료를 꺼내야 하듯, 코드 실행 전에 필요한 도구를 불러옵니다.
Python — 필수 임포트 
# ── 데이터 다루기 ──────────────────────────────
import pandas as pd
import numpy as np

# ── 머신러닝 (ML) ───────────────────────────────
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler, LabelEncoder, MinMaxScaler
from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor
from sklearn.linear_model import LogisticRegression, LinearRegression
from sklearn.metrics import accuracy_score, f1_score
from sklearn.metrics import mean_absolute_error, mean_squared_error, r2_score

# ── 딥러닝 (DL) ─────────────────────────────────
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense
from tensorflow.keras.callbacks import EarlyStopping
🔍 Step 1 — 데이터 로드 & 탐색 (EDA)
🗺️ 비유: 요리 전 재료 확인 "재료가 신선한지, 부족한 게 없는지" 먼저 확인하는 단계입니다.
Python — 데이터 탐색 공식 
# 파일 불러오기
df = pd.read_csv('파일명.csv')

# 기본 탐색 (항상 이 순서로!)
print(df.shape)          # (행 수, 열 수) → 데이터 크기 확인
print(df.head())         # 처음 5행 보기
print(df.info())         # 컬럼명, 타입, 결측치 개수 한눈에
print(df.describe())     # 숫자 컬럼 통계 (평균, 최솟값, 최댓값 등)
print(df.isnull().sum()) # 컬럼별 결측치(NaN) 개수
print(df.columns)        # 컬럼명 목록
🩹 Step 2 — 결측치 처리
🕳️ 비유: 빠진 퍼즐 조각 채우기 데이터에 구멍(NaN)이 있으면 모델이 오류를 냅니다. 구멍을 채우거나 제거해야 합니다.
Python — 결측치 처리 공식 
# 방법1: 결측치 있는 행 통째로 제거 (가장 간단)
df = df.dropna()

# 방법2: 숫자 컬럼 → 평균값으로 채우기
df['컬럼명'] = df['컬럼명'].fillna(df['컬럼명'].mean())

# 방법3: 숫자 컬럼 → 중앙값으로 채우기 (이상치 있을 때)
df['컬럼명'] = df['컬럼명'].fillna(df['컬럼명'].median())

# 방법4: 문자(범주) 컬럼 → 가장 많이 나오는 값으로 채우기
df['컬럼명'] = df['컬럼명'].fillna(df['컬럼명'].mode()[0])
💡
시험 팁: 문제에서 "결측치를 처리하시오"라고 하면 보통 dropna() 또는 fillna(mean())이 정답입니다.
🔤
따옴표는 작은따옴표(' ')와 큰따옴표(" ") 둘 다 OK!
파이썬에서 문자열은 두 가지 모두 완전히 동일합니다. 컬럼명 자리에는 실제 컬럼 이름을 넣으면 됩니다.

df['나이'] == df["나이"] ← 완전히 같습니다!

✔ 컬럼명을 모를 땐 먼저 df.columns 로 확인 후 그대로 복붙하세요. 대소문자·띄어쓰기까지 정확히 일치해야 합니다.
✂️ Step 2-2 — 이상치 제거 (조건 필터링 / 불리언 인덱싱)
🏫 비유: 시속이 300km/h 초과한 데이터 골라내기 drop()은 "이 행 번호를 지워라"는 방식이고,
조건 필터링은 "이 조건에 맞는 것만 남겨라"는 방식입니다. 결과는 같지만 코드가 훨씬 간결합니다!
Python — 조건 필터링 원리 (2단계로 쪼개서 이해) 
# ── 작동 원리 ──────────────────────────────────────────
# 단계①: 조건 확인 → 각 행에 True/False 딱지 붙이기
df.Speed_Per_Hour < 300
# 결과: 0     True
#       1     True
#       2     False  ← 이 행이 300 이상 (이상치!)
#       3     True

# 단계②: True인 행만 골라내기
df_temp = df[ df.Speed_Per_Hour < 300 ]
#          ↑전체df  ↑조건(안쪽) → True인 행만 통과!
Python — 두 가지 쓰는 방법 (완전히 같은 결과) 
# 방법1: 점(.) 표기법  ← 컬럼명에 띄어쓰기/특수문자 없을 때
df_temp = df[df.Speed_Per_Hour < 300]

# 방법2: 대괄호 표기법  ← 컬럼명에 띄어쓰기나 특수문자 있을 때 필수
df_temp = df[df['Speed_Per_Hour'] < 300]
Python — 자주 쓰는 조건 패턴 모음 
df[df['컬럼'] < 300]       # 300 미만만 남기기  (이상치 제거)
df[df['컬럼'] >= 0]         # 0 이상만 남기기   (음수 제거)
df[df['컬럼'] == '남']      # 특정 값만 남기기
df[df['컬럼'] != '기타']    # 특정 값 제외하기

# 조건 2개 동시 적용 (AND: &, OR: |)
df[(df['나이'] >= 20) & (df['나이'] <= 60)]  # 20 이상 AND 60 이하
df[(df['컬럼'] == 'A') | (df['컬럼'] == 'B')] # A 또는 B
⚠️
AND/OR 조건 사용 시 주의! 파이썬의 and/or가 아니라 반드시 & / | 를 쓰고, 각 조건을 괄호 ( )로 감싸야 합니다!
🗑️ Step 2-3 — drop() 옵션 완전 해설
📋 비유: 표에서 특정 열(컬럼) 또는 행을 가위로 잘라내기 drop()은 컬럼 또는 행을 삭제하는 함수입니다. inplace=True는 "원본 df를 바로 바꿔라"는 명령입니다. 없으면 원본은 그대로이고 결과만 반환합니다.
Python — drop() 기본 사용법 비교 
# ── inplace=False (기본값) ─────────────────────────────
# 원본 df는 그대로 유지되고, 삭제된 결과를 새 변수에 담아야 함
df_new = df.drop('RID', axis=1)
# df는 그대로, df_new에만 RID 컬럼이 없는 데이터가 담김

# ── inplace=True ──────────────────────────────────────
# 원본 df 자체를 바로 수정 → 새 변수 불필요
df.drop('RID', axis=1, inplace=True)
# 이제 df에서 RID 컬럼이 영구적으로 사라짐
첫 번째 인자 (labels)
삭제할 컬럼명 또는 행 인덱스
• 하나: 'RID'
• 여러 개: ['RID', 'ID', 'Name']
axis
삭제 방향
• axis=1 : 컬럼(열) 삭제 → 가장 많이 사용
• axis=0 : 행 삭제 (기본값)
inplace
원본 수정 여부
• True: 원본 df 자체를 바꿈 (반환값 없음)
• False: 원본 유지, 새 df 반환 (기본값)
errors
없는 컬럼명 삭제 시 처리
• 'raise': 오류 발생 (기본값)
• 'ignore': 오류 무시하고 넘어감
Python — drop() 활용 패턴 모음 
# 컬럼 1개 삭제 (inplace 방식)
df.drop('RID', axis=1, inplace=True)

# 컬럼 여러 개 동시 삭제
df.drop(['RID', 'ID', 'Name'], axis=1, inplace=True)

# 특정 행(인덱스) 삭제 → axis=0 또는 생략
df.drop(0, axis=0, inplace=True)     # 0번 행 삭제
df.drop([0, 1, 5], inplace=True)      # 0,1,5번 행 삭제

# 새 변수에 담는 방식 (inplace=False)
X = df.drop('정답컬럼', axis=1)         # 피처 분리 시 자주 사용!
💡
inplace 선택 기준:
• 원본 df를 이후에도 그대로 쓰고 싶다 → inplace=False (새 변수에 저장)
• 원본 df가 더 이상 필요 없고 바로 적용 → inplace=True
🔤 Step 3 — 범주형 인코딩 (문자 → 숫자)
🔢 비유: 언어 번역기 컴퓨터는 "남/여"같은 글자를 모릅니다. "0/1"로 번역해줘야 합니다.
Python — 인코딩 공식 
# 방법1: Label Encoding (순서가 있는 범주 또는 타겟 레이블)
#        예: "소/중/대" → 0/1/2
le = LabelEncoder()
df['컬럼명'] = le.fit_transform(df['컬럼명'])

# 방법2: One-Hot Encoding (순서 없는 범주)
#        예: "서울/부산/제주" → 각각 독립된 0/1 컬럼 3개 생성
df = pd.get_dummies(df, columns=['컬럼명'])
#                    ↑ drop_first=True 옵션 추가 가능 (다중공선성 제거)
✂️ Step 4 — 피처/레이블 분리 & 데이터 분할
📚 비유: 문제지와 답지 분리 X(피처) = 문제지, y(레이블) = 답지. 그리고 전체를 훈련용(train)과 시험용(test)으로 나눕니다.
Python — 데이터 분할 공식 (★암기 필수★) 
# ① 피처(X)와 레이블(y) 분리
X = df.drop('정답컬럼명', axis=1)  # 정답 컬럼 제외한 나머지
y = df['정답컬럼명']              # 정답 컬럼만

# ② train/test 분리
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,    # 20%를 테스트용으로 분리
    random_state=42  # 매번 같은 결과 → 고정값(보통 42 사용)
)

# 확인
print(X_train.shape, X_test.shape)
⚖️ Step 5 — 스케일링 (크기 통일)
📏 비유: 단위 통일 키(cm)와 몸무게(kg)의 단위가 다르면 모델이 혼란스럽습니다. 같은 범위로 통일시켜줍니다. 딥러닝, SVM, KNN에서는 필수이고, 트리 계열(랜덤포레스트)은 없어도 됩니다.
Python — 스케일링 공식 
# 방법1: StandardScaler — 평균 0, 표준편차 1 (z-점수 정규화)
#        언제? → 데이터가 정규분포에 가까울 때 (일반적으로 가장 많이 사용)
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)  # fit + transform 동시에 (훈련셋만)
X_test  = scaler.transform(X_test)       # transform만! (fit 하면 안 됨)

# 방법2: MinMaxScaler — 0~1 범위로 압축
#        언제? → 딥러닝, 범위를 0~1로 맞춰야 할 때
scaler = MinMaxScaler()
X_train = scaler.fit_transform(X_train)
X_test  = scaler.transform(X_test)
⚠️
절대 주의! X_test에는 반드시 .transform()만 사용하세요. .fit_transform()을 쓰면 정보 누수(data leakage)가 발생합니다!
🤖

머신러닝(ML) 분류 & 회귀 공식

모델 생성 → 학습 → 예측 → 평가 4단계 반복

🎯
분류란? 정답이 카테고리인 문제입니다. 예: "이 이메일은 스팸인가 아닌가?", "이 꽃은 무슨 종류인가?"
🌲 랜덤포레스트 분류 — 완전 공식 (★★★)
🌳 비유: 여러 전문가에게 의견 구하기 결정 트리(나무) 여러 그루를 만들어서 다수결로 결론을 냅니다. 가장 안정적이고 자주 출제됩니다!
Python — 랜덤포레스트 분류 전체 파이프라인 
# ① 모델 생성
model = RandomForestClassifier(
    n_estimators=100,   # 나무 개수 (기본값 100)
    random_state=42    # 재현성 고정
)

# ② 학습 (훈련 데이터로 배우기)
model.fit(X_train, y_train)

# ③ 예측 (테스트 데이터로 정답 맞추기)
y_pred = model.predict(X_test)

# ④ 평가
print("정확도:", accuracy_score(y_test, y_pred))
print("F1 Score:", f1_score(y_test, y_pred, average='weighted'))
#                                       ↑ 다중분류면 'weighted', 이진분류면 생략 가능

# ⑤ 특성 중요도 확인 (보너스)
print(model.feature_importances_)
🔵 로지스틱 회귀 분류

딥러닝(DL) Sequential 모델 공식

Dense 층 쌓기 → 컴파일 → 학습 → 예측

🏗️ Dense 레이어의 4가지 파라미터 — 완전 해설
🏢 비유: 회사 부서 설계하기 Sequential 모델 = 여러 부서(층)가 순서대로 일을 처리하는 회사.
Dense(128, activation='relu', input_shape=(10,)) = "128명이 일하고, relu방식으로 처리하며, 입력서류는 10장인 첫 번째 부서"
units (첫 번째 숫자)
뉴런(노드) 수
해당 층이 처리하는 특징의 수.
32, 64, 128, 256 등 2의 거듭제곱 사용.
보통 입력층~중간층: 크게, 출력층: 문제에 따라 고정.
activation
활성화 함수 — "어떤 방식으로 신호를 처리할까?"
• relu: 은닉층 기본값 (음수 → 0, 양수 → 그대로)
• sigmoid: 이진분류 출력층 (0~1 확률)
• softmax: 다중분류 출력층 (합계=1 확률)
• 없음(None/linear): 회귀 출력층
input_shape
입력 데이터 형상 — 첫 번째 층에만 필요!
피처(컬럼) 수 = X_train.shape[1]
형식: input_shape=(피처수,)
예: 10개 컬럼 → input_shape=(10,)
출력층 units 결정법
• 이진 분류: 1 (0 또는 1 확률 하나만)
• 다중 분류: 클래스 수 (예: 3종류 → 3)
• 회귀: 1 (하나의 숫자 예측)

🔵 입력층 (Input Layer)

Dense(128, activation='relu', input_shape=(피처수,))

→ 첫 번째 층만 input_shape 지정

→

🟡 은닉층 (Hidden Layer)

Dense(64, activation='relu')

→ relu가 기본. 층 수, 노드 수는 자유

→

🔴 출력층 (Output Layer)

Dense(?, activation='?')

→ 문제 유형에 따라 달라짐 (아래 표 참고)

⚫⚪ 딥러닝 이진 분류 (Binary Classification)
🚪 비유: 문이 열렸냐(1) 닫혔냐(0) 예측 정답이 2가지(예/아니오, 합격/불합격, 정상/이상)인 경우
Python — 이진 분류 Sequential 전체 공식 
model = Sequential([
    Dense(128, activation='relu', input_shape=(X_train.shape[1],)),
    #          ↑은닉층  ↑음수제거    ↑피처 수 자동 지정
    Dense(64,  activation='relu'),
    Dense(1,   activation='sigmoid')  # 출력: 1개 확률값 (0~1)
])

model.compile(
    optimizer='adam',           # 학습 최적화 알고리즘 (항상 adam)
    loss='binary_crossentropy', # 이진분류 전용 손실함수
    metrics=['accuracy']         # 학습 중 표시할 지표
)

early_stop = EarlyStopping(patience=5, restore_best_weights=True)
#                           ↑ 5번 연속 개선 없으면 조기 중단 (과적합 방지)

model.fit(
    X_train, y_train,
    epochs=100,            # 최대 반복 학습 횟수
    batch_size=32,         # 한 번에 학습하는 데이터 묶음 크기
    validation_split=0.2, # 훈련 데이터의 20%를 검증용으로
    callbacks=[early_stop]
)

# 예측 (0.5 기준으로 0 또는 1 분류)
y_prob = model.predict(X_test)
y_pred = (y_prob > 0.5).astype(int)
print("정확도:", accuracy_score(y_test, y_pred))
🌈 딥러닝 다중 분류 (Multi-class Classification)
🌺 비유: 꽃 종류 3가지 중 어느 것인지 예측 정답이 3가지 이상인 경우. 마지막 Dense의 units = 클래스 수!
Python — 다중 분류 Sequential 공식 
# 레이블을 One-Hot 인코딩으로 변환 (다중분류 필수!)
from tensorflow.keras.utils import to_categorical
y_train_cat = to_categorical(y_train)  # [0,1,2] → [[1,0,0],[0,1,0],[0,0,1]]
y_test_cat  = to_categorical(y_test)

num_classes = 3  # 클래스 수 (문제에 따라 변경)

model = Sequential([
    Dense(128, activation='relu', input_shape=(X_train.shape[1],)),
    Dense(64,  activation='relu'),
    Dense(num_classes, activation='softmax')  # 출력: 클래스 수만큼, 합계=1
])

model.compile(
    optimizer='adam',
    loss='categorical_crossentropy',  # 다중분류 전용 손실함수
    metrics=['accuracy']
)

model.fit(X_train, y_train_cat, epochs=100, batch_size=32, validation_split=0.2)

# 예측 (확률 중 가장 큰 인덱스가 예측 클래스)
y_prob = model.predict(X_test)
y_pred = y_prob.argmax(axis=1)

sparse_categorical_crossentropy 사용 시: to_categorical() 없이도 됩니다. loss='sparse_categorical_crossentropy'로 바꾸고 y_train 그대로 사용 가능!
📊 딥러닝 회귀 (Regression)
💰 비유: 아파트 가격을 숫자로 예측 마지막 Dense의 activation이 없고(None/linear), 출력 노드는 항상 1개!

    Dense(128, activation='relu', input_shape=(X_train.shape[1],)),
