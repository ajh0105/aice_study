빠른 참조

reset vs revert vs restore

명령어히스토리안전성사용 시점
git reset커밋 삭제 (재작성)⚠ 위험로컬 개인 작업
git revert새 커밋 추가 (보존)✓ 안전공유/원격 브랜치
git restore영향 없음✓ 안전파일 복원/unstage

fetch vs pull

명령어동작로컬 브랜치 영향
git fetch원격 다운로드만변경 없음
git pullfetch + merge 자동자동 병합

reset 옵션 비교

옵션커밋 취소스테이징 유지파일 내용 유지
--soft✓✓✓
--mixed (기본)✓✗✓
--hard✓✗✗

checkout vs switch vs restore

목적구버전신버전 (2.23+)
브랜치 전환git checkout <branch>git switch <branch>
브랜치 생성+전환git checkout -b <branch>git switch -c <branch>
파일 복원git checkout -- <file>git restore <file>
unstagegit reset HEAD <file>git restore --staged <file>
