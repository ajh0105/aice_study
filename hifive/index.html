    02 시스템별 역할 요약

    EDGE · FastAPI
    YOLO 결과 → 통과 판정 → gRPC 송신
    BACKEND · Spring Boot

    FRONTEND · Vue
    SPA + Pinia + 실시간 대시보드

    FastAPI Edge (fastapi-edge/)

    도로 카메라 옆에서 동작합니다. YOLOv8/OpenCV가 검출한 차량/번호판을 HTTP로 받고, 가상 통과선 알고리즘으로 ENTRY/EXIT을 판정한 다음 gRPC + Protobuf로 백엔드에 실시간 송신합니다. 백엔드가 죽어도 자체 큐에 쌓고 지수 백오프로 재연결합니다. OCR 신뢰도가 낮은 케이스는 단건 RPC로 분기해 검수 큐에 적재되도록 합니다.

    Spring Boot Backend (backend/)

    • REST: /api/auth/* 회원가입/로그인/로그아웃, /api/board 게시판/검수. Spring Session 기반 세션 쿠키 인증.
    • gRPC: FastAPI에서 보낸 PassageEventRequest를 수신·역직렬화·정산·DB 적재.

    Vue 3 Frontend (frontend/)

    • 랜딩(/)과 회사소개/솔루션/기술/도입안내/문의 정적 페이지
    • 회원가입/로그인 폼
    • 관리자 실시간 관제 대시보드(KPI · 통과 로그 · GPS · 게시판)
    • Pinia로 인증·게시판 상태, axios로 백엔드 호출, Vue Router 인증 가드

    3.1 FastAPI Edge — fastapi-edge/

│   ├── services/

│   │   ├── gps_service.py           트랙/차로별 GPS 캐시

    핵심 코드와 연결

    • proto/tolling.proto — service TollingEventService 안에 3개 RPC: SendPassageEvent(단건), StreamPassageEvents(양방향 스트림), Heartbeat. Edge·Backend stub의 단일 소스.
    • app/main.py — FastAPI(lifespan=...)로 부팅 시 CrossingLineDetector, GpsCache, TollingGrpcClient를 만들어 app.state에 보관, await grpc_client.start()로 채널·워커 기동.
    • app/api/routes.py — POST /v1/yolo/detections가 각 검출에 대해 detector.evaluate()로 통과 판정 → 통과면 grpc_client.enqueue_event() 호출. POST /v1/gps/track|lane/{id}는 GPS 캐시 갱신.
    • services/crossing.py — 트랙별 직전 중심좌표 보존, (직전→현재) 선분이 통과선 선분과 교차하면 통과로 판정. 외적 부호로 ENTRY/EXIT, 2초 cooldown, 5초 미관측 GC.
    • services/grpc_client.py — 내부 asyncio.Queue. 정상 트래픽은 StreamPassageEvents 스트림, 저신뢰 OCR(plate_confidence < 0.7)은 단건 SendPassageEvent로 분기. 채널 끊기면 0.5→1→2→4s 백오프 재연결, event_id(UUID)로 멱등성.
    • services/gps_service.py — 트랙 ID/차로 ID 기준 최신 GpsSample 보관 (스레드 안전).
    • core/config.py — BaseSettings로 EDGE_GRPC_TARGET, EDGE_LOW_OCR_CONFIDENCE_THRESHOLD, 통과선 좌표 4개 등 모든 튜닝 노브.

    핵심 코드와 연결

    • AuthController.signUp() → memberService.signUp(). 신규는 201 + AuthResponse(true,"가입완료"), 중복은 409 + AuthResponse(false,"이미 사용 중").
    • AuthController.login() → 성공 시 session.setAttribute("loginMember", member) 후 MemberResponse 반환. Vue 의 useAuthStore.login()이 data.member를 그대로 저장.
    • BoardController — GET은 모든 글, POST는 세션의 loginMember가 있으면 그 이름, 없으면 "방문자".
    • 현재 Service들은 in-memory. PostgreSQL/JPA로 교체 시 이 클래스만 갈아끼우면 됨.

    핵심 코드와 연결

    • App.vue — route.name === 'home'이면 글로벌 chrome 숨김. onMounted에서 useAuthStore().hydrate() 호출로 세션 복구.
    • router/index.js — beforeEach에서 requiresAuth면 비로그인 시 /login?redirect=...로, hideAuth면 이미 로그인 상태일 때 /로.
    • api/client.js — axios.create({ withCredentials:true }). dev에선 vite proxy /api → :8080.
    • stores/auth.js — state member, loading, error, getter isLoggedIn. login()은 호출 후 결과를 reactive에 반영하고 localStorage에 저장. hydrate()는 새로고침 후 복구.
    • stores/board.js — posts, lowConfidencePosts(신뢰도 0.7 미만 자동 필터). createdAt이 LocalDateTime 배열로 와도 ISO 문자열로 정규화.
    • views/DashboardView.vue — 시계/KPI/통과 로그 setInterval 시뮬레이션, GPS 패널, 게시판. 마운트 시 board.fetchAll().
    • views/HomeView.vue + views/home/* — 자식 섹션 5개 합성. 마운트 시 html.home-active 클래스 부여(scroll-snap 등 글로벌 영향), IntersectionObserver로 reveal/카운트업 트리거.

    04 데이터 / 호출 흐름 (시나리오)

    4.1 ★ 차량 통과 → 정산 (실시간 톨링 핵심 흐름)

    [카메라 + YOLOv8/OpenCV]
   │ 1. 프레임에서 차량/번호판 ROI 추출, OCR
   │ 2. 결과 1배치를 HTTP POST 로 엣지에 전송
   ▼
[FastAPI Edge]  POST /v1/yolo/detections   (app/api/routes.py)
   │ 3. routes.receive_detections()
   │      gps      = det.gps or gps_cache.latest_for(track_id, lane_id)
   │      decision = detector.evaluate(det)        # 통과선 판정
   │      if decision.crossed:
   │          await grpc_client.enqueue_event(det, decision, gps)
   │
   │ 4. enqueue_event() 가
   │      plate_confidence < 0.7  ⇒ asyncio.create_task(_send_unary(req))
   │      그 외 정상            ⇒ self._queue.put_nowait(req)
   ▼
[gRPC 채널]   localhost:9090   (Protobuf 바이너리, JSON 대비 -70%)
   │ 5. 정상: StreamPassageEvents 양방향 스트림
   │    저신뢰: SendPassageEvent 단건
   ▼
[Spring Boot Backend]
   │ 6. 수신·역직렬화 → DB 적재
   │ 7. plate / lane / direction / GPS 로 요금 계산
   │ 8. 저신뢰는 PENDING_REVIEW 상태로 검수 큐에 적재
   │ 9. PassageEventResponse(event_id, status) 응답
   ▼
[Vue 대시보드]
  10. /dashboard 에서 실시간 통과 로그·KPI 모니터링
  11. 저신뢰 로그는 amber 하이라이트, 보정 후 재전송 가능
    끊김에 강한 설계 — 6단계의 백엔드가 죽어 있어도 4단계의 큐는 계속 받고, _stream_worker가 0.5 → 1 → 2 → 4s 백오프로 재연결합니다. event_id(UUID)가 멱등성 키라 중복 도달도 백엔드가 거를 수 있습니다.

    06 실행 / 운영

    6.1 처음 띄우는 순서

    # 1) Spring Boot Backend  (REST + gRPC 9090)
cd backend
./gradlew bootRun                 # → http://localhost:8080

# 2) FastAPI Edge (별도 터미널)
cd fastapi-edge
python -m venv .venv && .\.venv\Scripts\Activate.ps1   # Windows PS
pip install -r requirements.txt
python scripts/generate_proto.py  # ★ proto stub 생성 (필수)
uvicorn app.main:app --reload --port 8000
# → http://localhost:8000/docs (Swagger), /healthz

# 3) Vue Frontend (또 다른 터미널)
cd frontend
npm install
npm run dev                       # → http://localhost:5173

    각 서비스는 다른 서비스가 떠 있지 않아도 부팅됩니다. 단 FastAPI는 백엔드(localhost:9090)가 없으면 gRPC 재연결 로그를 반복 출력하고, Vue는 /api/* 호출이 502가 됩니다. Vue mock UI(시계/KPI/통과 로그 시뮬레이션)는 백엔드 없이도 보입니다.
