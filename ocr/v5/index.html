🔍 기본 개념

OCR이란? 그리고 우리의 방식

OCR(Optical Character Recognition, 광학 문자 인식)은 이미지 속 글자를 컴퓨터가 읽을 수 있는 텍스트로 바꾸는 기술입니다. 우리 프로젝트의 목표는 단 하나 — 고속도로 CCTV가 찍은 자동차 번호판 사진을 보고 "01가1234" 같은 글자로 정확히 옮기는 것입니다.

비유로 이해하기. 사람이 번호판을 읽을 때는 ① 어디가 번호판인지 보고 ② 글자 모양을 눈에 담고 ③ 머릿속 글자 지식과 맞춰 ④ "01가1234"라고 말합니다. 우리 AI도 똑같은 4단계를 거칩니다. 차이는, 이 모든 걸 하나의 신경망이 한 번에 처리한다는 점입니다.

우리 OCR의 핵심 방식 — "End-to-End 딥러닝"

전통적인 OCR은 글자를 하나하나 잘라서(segmentation) 따로 인식했습니다. 하지만 번호판은 기울고, 흐리고, 글자 간격이 제멋대로라 "자르기"가 자주 실패합니다. 그래서 v5는 글자를 자르지 않고 번호판 전체를 통째로 읽는 방식을 씁니다. 이를 가능하게 하는 4가지 기술이 하나로 연결돼 있습니다.

📐

STN — 자세 교정

기울어지거나 비뚤어진 번호판을 신경망이 스스로 똑바로 펴줍니다. (Spatial Transformer Network)

👁️

CNN 백본 — 모양 보기

이미지에서 글자의 획·곡선 같은 시각적 특징을 뽑아냅니다. CBAM 어텐션으로 중요한 부분에 집중합니다.

🔗

BiLSTM — 순서 읽기

왼쪽에서 오른쪽으로 글자의 흐름과 앞뒤 맥락을 이해합니다. (양방향 순환 신경망)

✍️

CTC + 문법 — 정답 쓰기

글자 개수를 미리 정하지 않고도 "01가1234"를 출력하고, 한국 번호판 규칙으로 오류를 교정합니다.

💡 한 줄 요약

v5는 STN → CNN(CBAM) → BiLSTM → CTC가 한 몸으로 연결된 신경망입니다. 사진을 넣으면 글자가 나옵니다. 사람이 중간에 글자를 잘라줄 필요가 없습니다.

🛣️ 탄생 배경

왜 v5를 만들게 되었나

v5는 갑자기 나온 모델이 아닙니다. v1부터 v4까지 네 번의 시도에서 얻은 교훈을 모두 모아 만든 "기준 모델(reference model)"입니다. 이후 모든 버전이 v5를 출발점으로 삼습니다.

v1 — 베이스라인

완료 · 최초 구현

한국 번호판을 통째로 읽는 첫 end-to-end 모델. 입력 48×160, FocalCTC(γ=2.0). 중요한 규약 하나를 확립했습니다 — CTC 빈칸(blank)=0, 글자는 1번부터.

v2 — 해상도 실험 (중단)

미완료

입력을 48×160 → 32×128로 줄여 속도를 높이려 했지만, 작은 번호판에서 글자가 뭉개지는 치명적 문제 발견. 교훈: 해상도는 48×160 이상 유지.

v3 — 첫 완성 버전 (98.58%)

완료

실데이터+합성데이터 혼합으로 처음으로 쓸 만한 정확도 도달. 하지만 augmentation 라이브러리(albumentations) 호환 문제, 희귀 한글 학습 부족, 어려운 케이스 평가 없음이라는 숙제를 남김.

v4 — Mixup 제거 실험 (중단)

미완료

뚜렷한 개선이 없어 중단하고, 모든 교훈을 모은 v5 설계로 바로 넘어감.

v5 — 기준 모델 ★

완료 · Comp 0.9874

해상도 복귀(48×160), 라이브러리 의존 제거, 어려운/희귀 케이스 전용 평가, 문법 교정까지 — 그동안의 모든 교훈을 종합한 안정적 기준점.

✅ v5가 해결한 것

① 작은 번호판 → 해상도 복귀로 해결 · ② 라이브러리 호환 문제 → cv2만으로 재구현 · ③ 희귀 한글(바·배·사·아·자·허) → 가중 학습으로 정확도 100% 달성 · ④ 어려운 케이스 → 전용 평가셋 도입.

🧱 생성 과정

v5는 이런 7단계로 만들어집니다

"모델을 만든다"는 것은 코드 한 줄 실행이 아닙니다. 데이터를 정리하고, 어려운 시험문제를 따로 빼두고, 학습하고, 검증하고, 배포 형식으로 바꾸는 순서가 있는 작업입니다. 실제 실행 순서 그대로 정리했습니다.

글자 사전(vocab) 만들기

학습 데이터의 모든 라벨을 훑어 등장하는 글자를 모읍니다. 빈도 하위 30%와 지정 10자를 합쳐 "희귀 글자"도 정의합니다. (총 51종 글자)

python -m v5.vocab

어려운 시험지(hard val) 뽑기

작거나(≤40px), 흐리거나, 15° 넘게 기울거나, 희귀 글자가 있거나, 글자가 잘 안 보이는 사진을 골라 200~300장의 난이도 높은 평가셋을 따로 빼둡니다. 같은 번호판은 최대 1장만.

python -m v5.hard_split

데이터 나누기 (누수 방지)

학습용/검증용을 나눌 때 같은 번호판 글자가 양쪽에 동시에 들어가지 않도록 그룹 단위로 분리합니다. 시험 문제를 미리 공부하는 부정행위를 막는 장치입니다.

python -m v5.dataset

모델 학습 (단일 seed)

AdamW 옵티마이저로 최대 100 epoch 학습. 매 epoch마다 합성데이터 비율과 augmentation 강도를 조절하는 커리큘럼 학습을 적용합니다.

python -m v5.train --seed 42

최적 지점 선택 (composite)

단순 정확도가 아니라 0.5×어려운것 + 0.3×희귀것 + 0.2×전체로 계산한 종합 점수가 가장 높은 epoch을 저장합니다. 신뢰도(ECE)가 나빠지면 감점.

best.pt / last.pt

재현성 검증 (3-seed)

seed 42·3407·2026으로 세 번 돌려 "운이 좋아서"가 아님을 확인합니다. 평균 hard ≥ 0.987, 편차 ≤ 0.0008이면 통과.

python -m v5.run_3seeds

배포 형식으로 변환 (ONNX)

학습된 모델을 ONNX(opset 17, 동적 배치)로 내보내 TensorRT 같은 고속 추론 엔진에서 쓸 수 있게 만듭니다.

export_onnx(best.pt → best.onnx)

왜 글자를 안 잘라도 되나? CTC(Connectionist Temporal Classification)는 "40개의 작은 칸"에서 나온 예측을 보고, 빈칸과 중복을 알아서 정리해 최종 글자열을 만듭니다. 덕분에 글자가 몇 개인지, 어디서 끊기는지 사람이 알려줄 필요가 없습니다.
🔑 절대 바꾸면 안 되는 규약

⚙️ 처리 방식

실제로 번호판 한 장을 읽는 과정

학습이 끝난 v5가 새 사진 한 장을 받았을 때 일어나는 일을, 추론(inference) 코드 흐름 그대로 설명합니다.

① 전처리 — 사진을 모델이 좋아하는 형태로

잘라낸 번호판 영역을 48×160으로 리사이즈 → 색 순서를 RGB로 → 0~1로 나눈 뒤 (값−0.5)÷0.5로 정규화합니다. 모든 입력을 똑같은 기준으로 맞춰야 모델이 안정적으로 동작합니다.

② 신경망 추론 — 확률 지도 만들기

앞서 본 STN→CNN→BiLSTM→분류기를 통과하며, 40개 시점 × 51글자의 확률표가 만들어집니다. GPU에서는 AMP(자동 혼합 정밀도)로 더 빠르게 계산합니다.

③ 디코딩 — 확률을 글자로

단순히 매 시점 최고 확률 글자를 고르는 대신(greedy), beam search(폭 10)로 여러 후보 경로를 동시에 추적해 가장 그럴듯한 글자열을 찾습니다.

④ 문법 교정 — 한국 번호판 규칙 적용

한국 번호판은 "숫자 두세 자리 + 한글 한 자 + 숫자 네 자리" 형태입니다. 이 패턴을 통과하는 후보를 우선 선택하고, 다 실패하면 1순위 후보를 그대로 내보냅니다. 덕분에 "0이가1234" 같은 비정상 출력을 걸러냅니다.

# v5 추론 핵심 흐름 (V5Inference)
img = resize(crop, (48, 160))        # ① 크기 맞춤
img = (img/255 - 0.5) / 0.5           # ② 정규화 (RGB)
logits = model(img)                     # ③ 40×51 확률표
cands  = beam_search_decode(logits, w=10)  # ④ 후보 추적
text   = pick_valid_kr_plate(cands)       # ⑤ 문법 통과 후보 선택
# → "01가1234"
⚡ 배포 시

최종 모델은 ONNX(opset 17, 동적 배치)로 내보내 TensorRT 엔진으로 변환할 수 있어, 실시간 CCTV 스트림에서도 빠르게 동작합니다.

🗂️ 데이터 & 학습

무엇으로, 어떻게 가르쳤나

AI의 실력은 결국 데이터의 질과 학습 전략에서 나옵니다. v5는 깨끗한 실데이터를 중심에 두고, 합성데이터를 양념처럼 점진적으로 섞는 방식을 씁니다.

데이터 누수를 막는 그룹 분할

같은 번호판 글자(예: "01가1234")가 학습셋과 검증셋에 동시에 들어가면, 모델이 "외워서" 점수를 부풀립니다. v5는 번호판 글자열을 그룹 키로 삼아 한쪽에만 들어가도록 분할하고, 어려운 평가셋의 사진은 학습에서 3중으로 제외합니다.

커리큘럼 학습 — 쉬운 것부터 어려운 것까지

사람이 쉬운 문제부터 푸는 것처럼, 학습이 진행될수록 augmentation을 강하게, 합성데이터 비율을 점진적으로 조절합니다.

📊 성능

v5는 얼마나 잘 읽나

단일 지표가 아니라 5가지 각도로 평가합니다. 전체 정확도뿐 아니라 어려운 케이스, 희귀 글자, 헷갈리는 글자, 그리고 "모델이 자기 확신을 정확히 아는가(신뢰도)"까지 봅니다.

📐 신뢰도(ECE) = 0.0052

ECE는 "모델이 90% 확신할 때 실제로 90% 맞는가"를 재는 지표입니다. 0에 가까울수록 좋으며, v5의 0.0052는 매우 잘 보정된(well-calibrated) 수준입니다.

⚠️ 문제점 & 한계

v5가 아직 못하는 것들

정직한 평가가 다음 버전을 만듭니다. v5의 알려진 한계와 미완성 항목을 그대로 공개합니다.

📏

평가셋 기준 불일치

Hard 평가가 746장 기준이라 250장 기준의 후속 버전과 직접 비교가 어렵습니다. 공정 비교를 위한 기준 통일이 필요합니다.

📊

train accuracy가 근사값

현재 학습 정확도를 1 − 평균 train loss로 임시 계산합니다. 정확한 측정을 위해선 별도 평가 패스가 필요합니다.

🌫️

visible char ratio 휴리스틱

"글자가 잘 보이는 정도"를 Otsu 이진화의 단순 비율로 추정합니다. 더 정밀한 텍스트 마스크 기반 측정으로 개선 여지가 있습니다.

🎛️

INT8 양자화 미완

ONNX export는 제공하지만, TensorRT INT8 가속을 위한 보정(calibration) 데이터셋 500~1000장 추출은 아직 TODO입니다.

🏙️

도메인 합산의 위험

고속도로 극저해상도(ohjj) 데이터를 일반 데이터와 합쳐 학습하면 gradient가 교란됩니다. 도메인별 분리 학습이 필요합니다.

🗺️

지역명 번호판 미지원

"서울12가3456" 같은 지역명 포함 번호판은 패턴이 달라 아직 다루지 않습니다. detection 분기 방식이 필요합니다.

🧨 후속 버전이 겪은 교훈 (v5가 남긴 함정)

vocab 0-indexed 버그로 정확도가 0.53까지 폭락한 사례, EMA decay가 너무 높아(0.9995) 지표가 고착된 사례가 있었습니다. v5의 규약(blank=0, char 1-indexed)과 EMA 설정을 지키는 것이 안정성의 핵심입니다.

파인튜닝이 처음부터 학습보다 낫다

이후 경험에서 확인된 핵심 교훈입니다 — 이미 잘 수렴한 v5 가중치를 시작점으로 낮은 학습률(5e-5)로 파인튜닝하면, 처음부터 학습하는 것보다 빠르고 높은 성능에 도달합니다. v5는 그 출발점으로서 계속 가치를 가집니다.

🎯 핵심 메시지

v5는 "끝"이 아니라 모든 후속 실험의 안전한 기준선입니다. 새 아이디어가 v5보다 나은지 비교할 때, v5가 측정자(yardstick) 역할을 합니다.

어려운 단어, 쉽게 풀이

OCR이미지 속 글자를 컴퓨터 텍스트로 바꾸는 기술.
End-to-End입력(사진)에서 출력(글자)까지 하나의 신경망이 통째로 처리하는 방식.
STNSpatial Transformer Network. 기울거나 비뚤어진 이미지를 신경망이 스스로 펴주는 모듈.
CNN / 백본이미지에서 시각적 특징(획·곡선)을 뽑아내는 합성곱 신경망.
CBAM채널·공간 어텐션. 이미지에서 "어디를, 무엇을" 봐야 할지 강조하는 장치.
BiLSTM양방향 순환 신경망. 글자를 좌→우, 우→좌로 읽어 앞뒤 맥락을 이해.
CTC글자 개수를 미리 안 정해도 글자열을 출력하게 해주는 디코딩 방식. blank=0.
Beam Search여러 후보 글자열을 동시에 추적해 가장 그럴듯한 답을 찾는 탐색.
Augmentation원본 사진을 흐리게·기울게 변형해 학습 데이터를 늘리는 기법.
EMA학습 중 가중치의 이동 평균을 따로 유지해 더 안정적인 모델을 얻는 기법.
ECE모델의 확신 정도와 실제 정답률이 얼마나 일치하는지 재는 보정 오차.
ONNX / TensorRT학습 모델을 빠른 추론용 표준 형식·엔진으로 변환하는 도구.

Q & A

v5가 "기준 모델"이라는 게 무슨 뜻인가요?
v5는 가장 높은 점수의 모델은 아니지만, 안정적이고 재현 가능하며 모든 교훈이 녹아 있어 이후 모든 실험이 비교 대상으로 삼는 출발점입니다. 새 모델이 좋은지 나쁜지는 v5와 견줘 판단합니다.
왜 글자를 하나씩 자르지 않나요?
번호판은 기울고 흐리고 글자 간격이 일정치 않아 "자르기"가 자주 실패합니다. v5는 CTC 덕분에 자르지 않고 번호판 전체를 통째로 읽어, 자르기 실패로 인한 오류 자체를 없앱니다.
합성데이터는 왜 조금만 섞나요?
합성데이터는 다양성을 주지만 실제 분포와 다릅니다. 너무 많이 섞으면 현실과 멀어지므로, 실데이터를 중심에 두고 합성은 10~25%만 커리큘럼으로 점진 혼합합니다.
희귀 글자 100%는 어떻게 달성했나요?
바·배·사·아·자·허처럼 잘 안 나오는 한글이 포함된 샘플에 1.2배 가중치를 주고 손실 함수(FocalCTC)를 조정해 집중 학습한 결과입니다.
EMA를 ep10부터 켜는 이유는?
초반에는 가중치가 크게 출렁여서, 너무 일찍 평균을 내면 불안정한 값까지 섞입니다. 어느 정도 수렴한 ep10 이후부터 평균을 내야 안정적인 모델을 얻습니다. (파인튜닝 때는 오히려 EMA를 끄거나 약하게 합니다.)
v6.5가 더 좋은데 왜 v5를 배우나요?
v6.5는 폐기되었고, v5가 다시 현재의 기준 모델입니다. v5의 구조·학습 방식·규약을 이해하면 이후 어떤 버전이든 응용할 수 있습니다 — v5가 모든 것의 토대이기 때문입니다.
